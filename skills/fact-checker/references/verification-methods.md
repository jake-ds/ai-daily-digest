# 검증 방법론

## 1차: 원문 직접 대조

가장 기본적이고 신뢰할 수 있는 검증.

### 방법
1. 클레임에서 핵심 키워드 추출 (수치, 이름, 제품명 등)
2. 원문에서 해당 키워드 검색
3. 문맥 확인 — 같은 맥락에서 사용되었는지

### 판정 기준

**SUPPORTED**:
- 원문과 클레임이 의미적으로 일치
- 수치가 정확히 동일
- 인용이 원문과 동일

**PARTIAL**:
- 원문에 근거가 있지만 표현이 과장됨
  - 원문: "significant increase" → 초안: "3배 증가"
- 원문의 수치를 반올림/변형
  - 원문: "762" → 초안: "약 800개"
- 원문의 조건/맥락을 생략
  - 원문: "특정 조건에서 GPT-4를 넘었다" → 초안: "GPT-4를 넘었다"

**UNSUPPORTED**:
- 원문에 전혀 관련 내용 없음
- 초안에서 새롭게 창작한 수치/사실
- 다른 기사/출처의 정보를 원문에 있는 것처럼 서술

**UNVERIFIABLE**:
- 원문이 해당 주제를 다루지 않음
- 리서치 출처에서 가져왔으나 리서치 원문도 없음

## 2차: 리서치 결과 대조

원문에 없지만 리서치 결과에서 가져온 정보 검증.

### 방법
1. 리서치 결과에서 해당 정보 검색
2. 리서치 출처의 신뢰도 확인 (Tier 1-4)
3. 여러 출처에서 교차 검증

### 판정
- 리서치 Tier 1-2 출처에서 확인: SUPPORTED (출처 명시 권장)
- 리서치 Tier 3-4 출처에서만 확인: PARTIAL (교차 검증 권장)
- 리서치에서도 확인 안 됨: UNSUPPORTED

## 수정 제안 생성

### PARTIAL → 수정 방향
1. **과장 완화**: 구체적 수치를 원문 그대로 사용
2. **맥락 추가**: 생략된 조건/전제를 복원
3. **표현 정확화**: 의역을 원문에 가깝게 조정

### UNSUPPORTED → 수정 방향
1. **삭제 권고**: 원문에 근거 없는 내용 제거
2. **대체 제안**: 원문에서 비슷한 근거가 있으면 대체 표현 제안
3. **의견 표시**: "~로 보입니다" 등 저자 의견으로 전환

## 신뢰도 점수 계산

```
신뢰도 = (SUPPORTED × 1.0 + PARTIAL × 0.5) / 전체 클레임 × 100
```

| 점수 | 등급 | 의미 |
|------|------|------|
| 90-100 | 우수 | 거의 모든 클레임이 원문 기반 |
| 70-89 | 양호 | 대부분 정확, 일부 과장 |
| 50-69 | 주의 | 상당수 과장/미확인 |
| 0-49 | 위험 | 원문 반영 부족, 수정 필요 |
