# 클레임 유형 분류

## Type 1: 수치 클레임 (Numerical)

구체적 숫자가 포함된 주장.

**예시**:
- "764개 PR이 올라왔습니다"
- "가격은 기존 대비 80% 저렴합니다"
- "초당 464 토큰을 처리합니다"
- "3일 만에 10만 다운로드"

**검증 방법**: 원문에서 정확한 수치 대조. 반올림/추정치도 확인.

**주의**: "약 700개" vs "764개" — 원문이 "764"이면 정확히 써야 SUPPORTED.

---

## Type 2: 인용 클레임 (Quotation)

특정 인물의 발언을 인용한 주장.

**예시**:
- 'CEO가 "코드의 가치는 0으로 수렴한다"고 말했습니다'
- 'Andrew Ng 교수는 "기본기만 탄탄하면 충분하다"고 합니다'

**검증 방법**: 원문에서 해당 인물의 발언 확인. 따옴표 안 내용이 원문과 일치하는지.

**주의**: 의역 vs 직접 인용 구분. 의역이면 따옴표 없이 써야 함.

---

## Type 3: 비교 클레임 (Comparison)

두 대상을 비교하는 주장.

**예시**:
- "Claude가 GPT-4를 코딩 벤치마크에서 넘었습니다"
- "2시간 걸리던 작업이 3분으로 줄었습니다"
- "Supabase는 문서를 열고, SendGrid는 닫았습니다"

**검증 방법**: 원문에서 양쪽 대상에 대한 정보 확인. 비교의 근거가 있는지.

**주의**: 원문이 한쪽만 언급하고 다른 쪽은 저자가 추가한 경우 → PARTIAL.

---

## Type 4: 인과 클레임 (Causal)

원인과 결과를 연결하는 주장.

**예시**:
- "MCP 도입 이후 에이전트 연동이 증가했습니다"
- "파인튜닝 비용이 떨어지면서 경쟁이 치열해졌습니다"

**검증 방법**: 원문에서 인과 관계가 명시되어 있는지. 상관관계를 인과로 바꾸지 않았는지.

**주의**: 원문이 "이후(after)" 만 언급하고 "때문에(because)"는 아닌 경우 → PARTIAL.

---

## Type 5: 시기 클레임 (Temporal)

시점이나 기간에 대한 주장.

**예시**:
- "이번 주 발표된"
- "3월 출시 예정"
- "지난 분기 대비"

**검증 방법**: 원문에서 시기 정보 확인.

---

## Type 6: 존재/사실 클레임 (Existential)

특정 사실의 존재를 주장.

**예시**:
- "Cursor 사용자 데이터에 따르면"
- "Anthropic은 MCP 서버를 공식 출시했습니다"
- "오픈소스 도구가 좋아지면서"

**검증 방법**: 원문에서 해당 사실이 언급되는지.

---

## 클레임이 아닌 것 (검증 제외)

- 저자 의견: "저는 ~라고 봅니다", "제가 보는 시사점은"
- 일반론: "기술은 바뀌어도 원칙은 남습니다"
- 질문: "왜 이렇게 됐을까요?"
- 가정: "만약 ~라면"
- 미래 추측: "앞으로 ~할 것입니다" (단, 공식 발표 인용은 검증 대상)
