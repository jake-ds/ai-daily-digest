# Fact Checker

LinkedIn 초안에서 검증 가능한 클레임을 추출하고, 원문 콘텐츠와 대조하여 정확성을 검증하는 스킬.

## 워크플로우

```
초안 + 원문 → 클레임 추출 → 원문 대조 → 검증 결과 → 수정 제안
```

### Step 1: 클레임 추출

초안에서 검증 가능한 사실 주장(claim)을 추출한다.

클레임 유형은 [references/claim-types.md](references/claim-types.md) 참조.

**추출 대상**:
- 구체적 수치/통계: "764개 PR이 올라왔습니다"
- 직접 인용: 'Replit CEO가 "코드의 가치는 0으로 수렴한다"고 말했습니다'
- 비교 주장: "Claude가 GPT-4를 코딩 벤치마크에서 넘었습니다"
- 인과 주장: "MCP 도입 이후 에이전트 연동이 3배 증가했습니다"
- 시기 주장: "이번 주 발표된", "3월 출시 예정"

**추출 제외**:
- 저자의 의견/해석 ("저는 ~라고 봅니다")
- 일반론 ("기술은 바뀌어도 원칙은 남습니다")
- 질문문 ("왜 이렇게 됐을까요?")

### Step 2: 원문 대조

각 클레임을 원문 콘텐츠와 의미적으로 매칭한다.

검증 방법은 [references/verification-methods.md](references/verification-methods.md) 참조.

**검증 결과 분류**:

| 등급 | 의미 | 액션 |
|------|------|------|
| SUPPORTED | 원문에서 직접 확인됨 | 유지 |
| PARTIAL | 원문에 근거 있으나 과장/변형됨 | 수정 제안 |
| UNSUPPORTED | 원문에 근거 없음 (창작) | 삭제 또는 수정 |
| UNVERIFIABLE | 원문에 관련 내용 없어 검증 불가 | 경고 표시 |

### Step 3: 수정 제안

PARTIAL/UNSUPPORTED 클레임에 대해:
- 원문 기반 정확한 표현 제안
- 삭제 권고 (원문에 전혀 근거 없는 경우)
- 표현 완화 제안 (과장인 경우)

### Step 4: 검증 보고서

```
## 팩트 체크 결과

전체 클레임: 8개
- SUPPORTED: 5개
- PARTIAL: 2개
- UNSUPPORTED: 1개

### 상세

1. ✅ "764개 PR이 올라왔습니다" — 원문 확인됨
2. ⚠️ "3배 증가" → 원문은 "significantly increased" (구체적 수치 없음)
   → 수정 제안: "크게 증가했습니다"로 변경
3. ❌ "CEO가 직접 발표" — 원문에 해당 내용 없음
   → 삭제 권고

신뢰도 점수: 75/100
```

## 판단 원칙

1. **원문이 최고 권위**: 원문에 있는 내용만 팩트로 인정
2. **수치는 정확히**: 원문의 숫자를 바꾸거나 반올림하면 PARTIAL
3. **인용은 그대로**: 직접 인용은 원문과 정확히 일치해야 SUPPORTED
4. **추론 허용**: "이는 ~를 의미합니다" 같은 해석은 클레임이 아님
5. **리서치 출처도 검증**: 웹 리서치에서 가져온 정보도 대조 대상
