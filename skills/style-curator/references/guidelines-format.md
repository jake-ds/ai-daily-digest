## 각 시나리오 섹션 필수 항목

1. **언제 사용** — 어떤 콘텐츠 유형에 적합한지
2. **훅 스타일** — 패턴 + 예시
3. **본문 구조** — 넘버링된 단계
4. **마무리 스타일** — 패턴 + 예시

## 예시 섹션 규격

- 코드 블록(```)으로 감싸기
- 실제 사용 가능한 완성된 포스트
- 원문 링크 자리 표시: `원문: https://...`
- 1800-2800자 범위 준수

## 가이드라인 갱신 원칙

1. **추가 우선**: 기존 내용 삭제보다 새 패턴 추가 선호
2. **예시 교체**: 오래된 예시를 최신 승인 포스트로 교체
3. **금지 사항 갱신**: 반복 실패하는 패턴을 금지 목록에 추가
4. **시나리오 확장**: 새로운 시나리오가 반복되면 기존에 추가
5. **버전 관리**: 변경 시 backup 생성 (`data/linkedin_guidelines.md.backup`)

## StyleBriefBuilder 파싱 의존성

guidelines.md를 파싱하는 정규식 패턴:
- Persona: `## Persona\n(.*?)(?=\n---)`
- 시나리오: `### 시나리오 {X}:.*?(?=\n---|\n### 시나리오 [A-F]:|$)`
- 공통 규칙: `## 공통 규칙\n(.*?)(?=\n## |$)`
- 예시: `### 시나리오 {X} 예시.*?```\n(.*?)````

이 정규식이 깨지지 않도록 구조를 유지해야 한다.
