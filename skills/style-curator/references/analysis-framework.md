# 포스트 분석 프레임워크

## 분석 차원

### 1. 톤 분석

| 요소 | 측정 방법 |
|------|----------|
| Formality | 하십시오체 vs 해요체 비율 |
| Persona voice | Jake의 특성 표현 사용 빈도 ("저는 ~라고 봅니다", "패턴이 보입니다" 등) |
| 겸손도 | 조언톤 vs 경험공유톤 비율 |
| 확신도 | 단정 표현 vs 유보 표현 비율 |

### 2. 구조 분석

| 요소 | 측정 방법 |
|------|----------|
| 훅 패턴 | 숫자/질문/대비/공감/발견 중 어떤 유형 |
| 본문 흐름 | 점진적 논증 vs 병렬 나열 |
| 넘버링 사용 | 1,2,3 구조 사용 여부 및 빈도 |
| 구분선 사용 | ㅡ 구분선 사용 위치와 빈도 |
| 마무리 패턴 | 경험연결/행동선언/고백/원칙/결정/격려 중 어떤 유형 |
| 단락 수 | 평균 단락 수 |

### 3. 어휘 분석

| 요소 | 측정 방법 |
|------|----------|
| 선호 표현 | 자주 사용되는 구절 빈도 |
| 금지 표현 | 사용된 금지 표현 감지 |
| 전문용어 | 영어 전문용어 사용 패턴 |
| 인용구 | 따옴표 인용 빈도 및 패턴 |

### 4. 문장 분석

| 요소 | 측정 방법 |
|------|----------|
| 평균 문장 길이 | 글자 수 기준 |
| 장단문 비율 | 20자 미만(단문) vs 50자 이상(장문) 비율 |
| 도치문 빈도 | "~뭘까요", "~보입니다" 패턴 |
| 질문문 빈도 | 물음표 사용 빈도 |

## 패턴 추출 규칙

### Positive 패턴 (효과적)
- 사용자가 승인한 draft에서 반복되는 구조
- 평가 점수 80+ draft의 공통 특성
- 사용자 피드백에서 긍정 언급된 요소

### Negative 패턴 (피해야 할)
- 평가에서 FAIL된 항목이 2회 이상 반복
- 사용자 피드백에서 수정 요청된 요소
- 금지 표현 사용 패턴
