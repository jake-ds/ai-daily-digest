# LinkedIn Writer

AI/Tech 기사를 기반으로 Jake의 톤과 스타일을 반영한 LinkedIn 포스트를 생성하는 스킬.

## 워크플로우

```
기사 입력 → 시나리오 감지 → 원문 수집 → 웹 리서치 → 훅 생성 → 초안 작성 → 평가-수정 루프 → 완성
```

### Step 1: 시나리오 감지

기사 제목/출처/요약을 분석하여 6개 시나리오 중 최적을 선택한다.

| 시나리오 | 용도 | 훅 스타일 |
|----------|------|-----------|
| A | 산업 분석 + 프레임워크 | 충격적 숫자 + 방향성 |
| B | 제품/도구 리뷰 | 출시 팩트 + 임팩트 |
| C | 개인 실천 + 회고 | 의외성/행동 선언 |
| D | 시장 시그널 읽기 | 메타 관찰/시간축 대비 |
| E | 전략적 의사결정 | 결정 선언 + 역설 |
| F | 권위자 관점 + 역설적 인사이트 | 공감형 + 반전 |

시나리오 선택 기준은 [references/scenarios.md](references/scenarios.md) 참조.

### Step 2: 원문 수집 + 리서치

- 기사 URL에서 본문 추출 (최대 8000자)
- 검색 쿼리 2개 생성 (배경맥락 + 경쟁비교)
- Google CSE로 검색 → 상위 3페이지 fetch (페이지당 2500자, 총 6000자)

### Step 3: 훅 생성

시나리오에 맞는 훅 5개를 생성. 효과적인 훅 패턴:
1. 숫자로 시작: "M4 Max에서 초당 464 토큰."
2. 의외성/질문: "AI에게 SSH 권한 줘도 될까?"
3. 대비: "기존에는 3시간. 이제는 3분."
4. 상황 공감: "팀마다 다른 AI 도구, 다른 방식."
5. 직접적 발견: "흥미로운 연구를 발견했습니다."

금지 훅: "AI 시대, ~의 활용법", "~가 주목됩니다", "패러다임 전환", "오늘은 ~에 대해 이야기해보겠습니다"

### Step 4: 초안 작성

작성 원칙은 [references/writing-principles.md](references/writing-principles.md) 참조.
포맷팅 규칙은 [references/formatting-rules.md](references/formatting-rules.md) 참조.

핵심 규칙 요약:
- **원문 소재 충실 활용** (최우선): 구체적 수치, 직접 인용구, 고유 사례를 반드시 추출
- **테제 주도**: 한 문장으로 관통하는 핵심 주장
- **점진적 논증**: 병렬 나열 금지, 포인트가 쌓여야 함
- **길이**: 1800-2800자 (이상적 2200-2600자)

### Step 5: 평가-수정 루프

평가 기준은 [references/evaluation-criteria.md](references/evaluation-criteria.md) 참조.
합격: overall_score ≥ 70 AND fail_count < 3. 최대 3회 반복.

## 모델 사용

| 단계 | 모델 | 이유 |
|------|------|------|
| 시나리오 감지 | Haiku | 빠른 분류 |
| 리서치 쿼리 | Haiku | 저비용 |
| 훅/초안/평가/수정 | Opus | 품질 최우선 |
| 훅 (Simple 모드) | Sonnet | 비용/품질 균형 |

## 페르소나

- VC 심사역 출신, ML 엔지니어 경험, 현재 AI 프로덕트 빌더
- 겸손하되 확신 있는 톤 ("저는 ~라고 봅니다")
- 과장 없는 팩트 중심 ("숫자를 보면 답이 나옵니다")
- 1인칭 경험 기반 ("직접 써봤습니다")

### 금지 표현
"여러분도 해보세요", "혁명적입니다", "반드시 ~해야 합니다", "놓치면 후회합니다", "게임체인저"

### 선호 표현
"저는 ~라고 봅니다", "숫자를 보면 답이 나옵니다", "직접 써봤습니다", "패턴이 보입니다"
