# LinkedIn 포맷팅 규칙

## 문체

- **기본**: 하십시오체 ("~입니다", "~했습니다")
- **리듬 전환**: 해요체로 가끔 변화 ("~해요", "~네요")
- **자연스러운 톤**: 발표가 아닌 대화처럼
- 짧은 문장과 긴 문장을 섞어 자연스러운 리듬감 (단문만 반복하면 AI스러움)
- 도치문("~뭘까요.", "~보입니다.")은 글 전체에서 1-2회만 허용

## 구조

- 줄바꿈으로 단락을 명확히 구분
- 넘버링(1, 2, 3)을 활용하여 가독성 향상
- 구분선(ㅡ)을 활용하여 시각적 정리

## 길이

- 최소: 1,800자
- 최대: 2,800자
- 이상적: 2,200~2,600자

## 포스트 구조

1. **훅** (1-2문장): 스크롤을 멈추게 하는 첫 문장
2. **본문** (3-4단락): 시나리오별 구조 따르기
3. **마무리** (1-2문장): 개인적 시선 또는 열린 질문
4. **링크**: 원문 URL

## 출력 형식

- 제목/헤더 없이 본문만 출력. 첫 줄이 곧 훅
- 설명이나 주석 없이 바로 사용 가능한 형태
- 마지막에 원문 링크 한 줄

## 금지 사항

- 조언톤: "~하세요", "~해보세요" → "저는 ~합니다"
- 공포 마케팅: "지금 안 하면 뒤처집니다"
- 이모지 사용
- "여러분" 호칭
- 과장 표현: "혁명", "패러다임 시프트", "게임체인저"
- 형식적 마무리: "읽어주셔서 감사합니다"

## 권장 사항

- 숫자와 구체적 사례 사용
- 1인칭 경험 중심
- 질문으로 끝내기 (선택적)
- 단락 구분 명확히
- 원문 링크 포함
- 권위자 인용으로 신뢰성 확보
- 인용구 강조 (따옴표 활용)
- 메타포를 통한 개념 설명

## 마무리 스타일 종류

1. **본인 경험 연결**: "저도 이 방향으로 실험을 시작했습니다."
2. **가벼운 행동 선언**: "이번 주말에 더 깊게 파볼 예정입니다."
3. **자기 고백**: "솔직히 불편했습니다. 하지만 필요한 불편함이었습니다."
4. **변하지 않는 원칙**: "기술은 바뀌어도 문제 해결 능력의 가치는 변하지 않습니다."
5. **최종 결정**: "단기 성과보다 장기 학습을 선택했습니다."
6. **격려형**: "대부분이 아직 시작하지 않았습니다. 지금의 FOMO는 기회가 남아있다는 신호입니다."
