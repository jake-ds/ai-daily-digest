# 포스트 평가 기준

## 정규식 기반 사전 검증 (LLM 불필요)

| 항목 | 규칙 | 패널티 |
|------|------|--------|
| 글자수 | 1800-2800자 | FAIL |
| 금지 표현 | "여러분", "해보세요", "혁명적", "패러다임", "게임체인저" | FAIL |
| 조언톤 | "~하세요", "~해보세요" 패턴 | FAIL |
| 공포 마케팅 | "뒤처", "놓치면" | FAIL |
| 이모지 | 이모지 문자 감지 | FAIL |
| 원문 링크 | URL 포함 여부 | FAIL |
| 단락 수 | 최소 3개 | WARNING |
| 훅 길이 | 첫 줄 100자 이내 | WARNING |

## AI 기반 평가 (LLM 사용)

JSON 형식으로 평가 결과를 생성:

```json
{
  "overall_score": 78,
  "items": [
    {
      "category": "원문 활용",
      "rule": "원문의 구체적 수치/인용구 사용",
      "pass": true,
      "comment": "3개의 구체적 수치를 원문에서 정확히 인용"
    },
    {
      "category": "테제",
      "rule": "한 문장으로 관통하는 핵심 주장",
      "pass": true,
      "comment": "명확한 테제 제시됨"
    }
  ],
  "summary": "원문 활용도 높고 논증 구조가 탄탄하나 마무리가 다소 약함"
}
```

## 합격 기준

- `overall_score` ≥ 70
- FAIL 항목 < 3개
- 두 조건 모두 충족 시 합격

## 평가-수정 루프

1. 초안 생성
2. 정규식 검증 → FAIL이면 즉시 수정
3. AI 평가 → score/fail_count 확인
4. 불합격 시 FAIL 항목만 타겟으로 수정 프롬프트 생성
5. 수정된 초안으로 재평가
6. 최대 3회 반복 (Agent 모드) / 2회 (Simple 모드)

## 평가 차원

| 차원 | 평가 내용 |
|------|----------|
| 원문 활용 | 구체적 수치/인용구가 원문에서 추출되었는가 |
| 테제 | 핵심 주장이 한 문장으로 명확한가 |
| 논증 구조 | 점진적으로 쌓이는가 (병렬 나열 아닌가) |
| 구체성 | 이름/숫자로 대비가 있는가 |
| 톤 | Jake의 페르소나와 일치하는가 |
| 포맷 | 길이, 단락, 훅, 마무리 규격 준수 |
| 금지 사항 | 금지 표현/패턴 위반 없는가 |
